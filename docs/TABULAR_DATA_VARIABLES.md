# 머신러닝 테이블형 데이터(Tabular Data) 변수 유형 가이드북

본 문서는 머신러닝(ML) 및 데이터 사이언스에서 가장 기본이 되는 **테이블형 데이터의 변수(Feature) 유형**과 이를 모델 학습에 활용하기 위한 **전처리 전략**을 명세합니다.

---

## 1. 변수 분류 체계도

테이블형 데이터의 변수는 크게 수학적 연산 가능 여부에 따라 **범주형(Categorical)**과 **수치형(Numerical)** 두 가지로 분류됩니다.

```text
테이블형 데이터 변수 (Variables)
 ├── 1. 범주형 변수 (Categorical) - 질적 데이터
 │    ├── ① 명목형 변수 (Nominal) ── 예: 원료 물질명, 전해질 종류, 성별
 │    └── ② 순서형 변수 (Ordinal) ── 예: 안전 등급, 만족도, 학점
 │
 └── 2. 수치형 변수 (Numerical)   - 양적 데이터
      ├── ① 이산형 변수 (Discrete) ── 예: 사이클 횟수, 불량 개수, 자녀 수
      └── ② 연속형 변수 (Continuous) ─ 예: 방전 용량, 전압, 온도, 몸무게
```

---

## 2. 세부 변수 유형 설명

### 1) 범주형 변수 (Categorical Variables)
형태가 숫자이더라도 크기 차이나 사칙연산의 물리적 의미가 없으며, 대상을 **단순히 분류(Classification)**하기 위한 변수입니다.

#### ① 명목형 변수 (Nominal Variables)
*   **정의**: 순서, 우열, 혹은 상하 관계가 전혀 없는 순수한 범주입니다.
*   **배터리 데이터 예시**: 양극재 구조(`material_structure`), 리튬 원료(`Li_source`), 전해질 종류(`electrolyte`)
*   **일반 예시**: 성별(남/여), 혈액형(A/B/O/AB), 거주 국가(한국/미국/일본)
*   **머신러닝 전처리**: 
    *   **원-핫 인코딩(One-Hot Encoding)**: 고유값 개수가 적을 때 사용 (가장 보편적)
    *   **타겟 인코딩(Target Encoding)**: 고유값 개수가 매우 많을 때 사용 (과적합 주의)

#### ② 순서형 변수 (Ordinal Variables)
*   **정의**: 사칙연산은 불가능하지만, 값들 사이에 명확한 **순서나 등급(Rank)**이 존재하는 변수입니다.
*   **배터리 데이터 예시**: 셀 품질 등급 (Grade A > Grade B > Grade C)
*   **일반 예시**: 설문조사 만족도 (1: 매우 불만족 ~ 5: 매우 만족), 학점 (A > B > C), 옷 사이즈 (S < M < L)
*   **머신러닝 전처리**: 
    *   **레이블 인코딩(Label Encoding)**: 순서적 의미를 반영하여 자연수 값(1, 2, 3...)으로 매핑 처리

---

### 2) 수치형 변수 (Numerical Variables)
값의 수치적 크기 자체가 의미를 가지며, **사칙연산(덧셈, 뺄셈 등)이 수학적으로 유효**한 변수입니다.

#### ① 이산형 변수 (Discrete Variables)
*   **정의**: 소수점으로 나누어지지 않고 하나씩 **셀 수 있는(Countable) 정수 형태**의 변수입니다.
*   **배터리 데이터 예시**: 배터리 수명 테스트의 총 충방전 사이클 횟수 (예: `150`회, `300`회)
*   **일반 예시**: 차량 보유 대수(대), 가구원 수(명), 공장의 불량품 개수(개)
*   **머신러닝 전처리**: 
    *   대부분 수치형 값 그대로 사용
    *   값의 편차가 극도로 큰 경우 로그 변환(Log Transform) 적용

#### ② 연속형 변수 (Continuous Variables)
*   **정의**: 정수 사이에 무한히 많은 실수가 존재하며 소수점 단위로 쪼갤 수 있는 **연속적인 측정값**입니다.
*   **배터리 데이터 예시**: 잔류 방전 용량(`remain_capacity`), 전압 범위(`voltage_range`), 내부 저항, 온도
*   **일반 예시**: 키(cm), 몸무게(kg), 주식 가격, 시간
*   **머신러닝 전처리**:
    *   **표준화(Standardization)**: 평균 0, 분산 1로 변환 (경사하강법 기반 모델에 필수)
    *   **정규화(Normalization)**: 값을 0과 1 사이로 압축 (Min-Max Scaling)

---

## 3. 머신러닝 특수 변수 유형

> [!NOTE]
> 실무 테이블 데이터에는 위 기본 분류 외에도 전처리 시 특별한 주의를 요구하는 데이터 타입이 존재합니다.

*   **이진 변수 (Binary Variables)**:
    *   오직 두 가지 상태만 존재하는 변수로 명목형의 특수 사례입니다. (예: 불량 여부 `0` 혹은 `1`, 로그인 여부 `True` 혹은 `False`)
*   **날짜/시간 변수 (DateTime Variables)**:
    *   `2026-05-22 15:00:00`과 같은 포맷입니다. 
    *   모델이 직접 학습할 수 없으므로, 이를 분해하여 **'요일', '시간대', '연도별 분기', '휴일 여부'** 등의 독립적인 파생 변수로 변환해야 합니다.
*   **텍스트 / 비정형 변수 (Text / Unstructured Variables)**:
    *   비고란 메일 주소나 자유 서술형 메모입니다.
    *   NLP 기술이나 단어 벡터화(TF-IDF, Embedding)를 거쳐 수치 행렬로 다룹니다.

---

## 4. 변수 유형별 머신러닝 모델 추천 및 영향도

| 변수 유형 특징 | 딥러닝 (MLP/DNN) | 트리 계열 (LGBM, CatBoost) | 선형 모델 (Linear Reg.) |
| :--- | :--- | :--- | :--- |
| **범주형(Categorical) 변수 중심** | 원-핫 인코딩 필수, 오버피팅 주의 | **CatBoost** 사용 시 자동 처리 극대화 | 더미 변수화 필수 |
| **수치형(Continuous) 스케일 차이** | **극도로 민감** (스케일링 필수) | **영향 없음** (트리 분할 방식은 스케일에 영향받지 않음) | **민감** (정규화 권장) |
| **결측치(Missing Value) 존재 시** | 학습 불가능 (대체 필수) | **자체 내장 처리 가능** | 학습 불가능 (대체 필수) |
